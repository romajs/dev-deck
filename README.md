# DevDeck

DevDeck is a native macOS utility for discovering local development servers, the ports they use, and the project metadata behind them.

It is intentionally lightweight:

- Swift + SwiftUI
- macOS 14+
- native window app with optional `MenuBarExtra` status bar icon
- no Electron
- no webview
- no telemetry

## Status

DevDeck is early-stage source-available software. The current app is useful for local development visibility, but the UX and distribution story are still evolving.

The source is visible for review and reference, but DevDeck is not open source under an OSI-approved license. The license does not grant permission to use, copy, modify, redistribute, commercialize, or create derivative works without written permission.

GitHub Releases provide downloadable universal macOS app bundles for Apple Silicon and Intel Macs. Release bundles are ad-hoc signed but not notarized yet. There is no Homebrew cask yet. Downloadable release artifacts do not grant permission beyond the repository license.

## Features

- Detects local listening TCP ports via `lsof`
- Shows PID, command, user, CPU, RAM, uptime, and cwd when available
- Detects development apps across Node.js, Deno, Bun, Python, Ruby, Java/JVM, Go, Rust, PHP, .NET, and Elixir
- Reads project manifests such as `package.json`, `pyproject.toml`, `Gemfile`, `pom.xml`, `Cargo.toml`, `go.mod`, `composer.json`, `mix.exs`, and `.csproj`
- Detects common frameworks such as Next.js, Vite, NestJS, Django, Flask, FastAPI, Rails, Spring Boot, Laravel, ASP.NET Core, Phoenix, and more
- Reads Git branch, dirty status, and remote origin URL
- Shows per-runtime badges and per-runtime port settings
- Provides an optional macOS status bar icon that opens the main window and compact process list
- Opens detected apps in the browser
- Terminates a selected process with confirmation
- Auto-refreshes with conservative polling and pauses when the app window/popover is not visible
- Configures ignored process names and support paths to reduce false positives from background apps

## Build

Build the native app bundle:

```sh
Scripts/build-app.sh
open .build/DevDeck.app
```

`Package.swift` is included for source indexing and SwiftPM-compatible toolchains, but the supported MVP distribution artifact is the `.app` bundle generated by `Scripts/build-app.sh`.

Package a local release zip:

```sh
Scripts/package-release.sh local
```

Pull requests and pushes to `main` run CI. Public releases are published only when a version tag such as `v0.2.0` is pushed, or when the release workflow is manually run for an existing tag.

## Product Notes

- The status bar icon is optional and intentionally does not show a live badge count; process data loads when a visible surface is active.
- Port conflict detection and port-change suggestions are future backlog items, not current app features.
- Dev ports are configured per runtime in Settings. These ports are used for candidate detection.
- Ignored process names and support paths are editable in Settings. Defaults include common macOS support locations such as `~/Library`, `/Library`, `/System`, `/Applications`, and temporary support folders. Reset updates the draft fields; Save applies them.
